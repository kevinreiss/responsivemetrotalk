## Fluid Media 1. 100% Width/Height: Auto 2. Best Strategy for Now 3. Flexible is the least mature of responsive tools
## Media Queries * Apply a different set of CSS rules based on the size of the screen size * Operate on: Min/Max Width Values * Page Orientation: Landscape v. Portrait * You use them to invoke "breakpoints" - the point at which your content "responds" * To invoke them you can use generally accepted [defaults](http://twbs.github.io/bootstrap/css/#grid) * Or - [build your own](http://breakpoint-sass.com/)

The Standard Media Query Strategy

  1. One base default media Query (In Bootstrap it's the smallest size)
  2. Styles to be applied when matched by different media queries are wrapped within that query
  3. Can get rather unwieldy on large projects with a lot of overrides.

## CSS Pre-Processors 1. Compile your CSS automatically 2. [SASS](http://sass-lang.com/) 3. [Less](http://lesscss.org/) 4. Just use one or the other 5. I'll talk about SASS

You Will Need A Functional Ruby Environment

Sass is a ruby gem

> gem install sass

SASS Features

  • Variables
  • Functions - built-in helpers for common CSS tasks
  • Nesting Groups of Selectors
  • Reusable Components of CSS - Mixins
  • Low learning curve - Syntax is very similar to CSS
  • .scss file extension

SASS Benefits

Four big wins

  1. Large projects become more maintainable and readable
  2. Re-use of CSS can go beyond copy and paste
  3. Repetitive and verbose tasks become simple (think image sprites)
  4. Leverage the power of a full programming language to manage grids and breakpoint calculations

Compass - An Essentail Sass Development Tool

  • Compass is also a ruby gem. See config.rb in a SASS project directory
  • > gem install compass
  • Compass compiles your SASS to CSS for use in the browser
  • Provides many extensions for common styling, typography, and CSS features
  • Never write a vendor prefix (webkit-,-ms, etc.) again

Using Sass with Zen Grids

  • Zen-grids is a compass extension and a Ruby Gem
  • > gem install zen-grids
  • It allows you to define complex, "semantic" grids
  • Some Examples

  • When integrated with the breakpoint compass extension it can easily support complex media queries.
